ncar-nsf-neon

A containerized JupyterLab environment for running CTSM (Community Terrestrial Systems Model) simulations against NEON observational data — with built-in Kalman-filter calibration, PFT perturbation experiments, and S3-backed forcing pipelines.

Launch the Tool → View on GitHub

What this is

A reproducible analysis environment for the CTSM-Lab community. It packages NCAR's cesm-lab-neon image plus our own analytics utilities and notebooks so anyone can run NEON-site simulations and calibrate them against tower observations without manually building the CTSM toolchain.

CTSM simulations

One-command transient runs for any of 47 NEON sites via run_neon_v2.py, with optional S3-backed forcing data and counterfactual transforms (scaling, seasonal, noise).

PFT perturbation

Shift cover between plant functional types in the surface dataset and compare ecosystem-level responses to a control run.

Calibration & misfit

Kalman-filter calibration of model output against observations, with built-in residual diagnostics.

Interactive hubs

Guided notebooks for data exploration (Data Hub), experimental design (Design Hub), and modeling (Modeling Hub).

How to access

If you're a user: click Launch the Tool above, sign up with a username + password, and an admin will approve your account. Once approved you'll get your own JupyterLab session in a fresh CTSM container.

If you'd rather run it locally:

docker run --rm -p 8888:8888 ghcr.io/mariaob1201/ncar-nsf-neon:latest

Then open http://127.0.0.1:8888/lab. The image is ~2 GB; first pull takes a few minutes.

Tech stack

CTSM 5.1 JupyterLab JupyterHub xarray Docker GHCR GitHub Actions CI Caddy / Let's Encrypt

Source code, Dockerfile, and GitHub Actions CI all live in the public repo. Container images are published to ghcr.io/mariaob1201/ncar-nsf-neon on every push to main.

For developers

To run a self-hosted JupyterHub instance using this image (the same setup that powers the Launch button above):

git clone https://github.com/mariaob1201/ncar-nsf-neon.git
cd ncar-nsf-neon/deploy/jupyterhub
cp .env.example .env
# edit JUPYTERHUB_ADMIN_USERS in .env
docker compose up -d --build

Full walkthrough including HTTPS via Caddy: deploy/jupyterhub/README.md.
